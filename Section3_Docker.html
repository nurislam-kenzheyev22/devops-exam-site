Q18: Why do we use multi-stage Docker builds?

Answer (English):

Multi-stage builds reduce final image size by excluding build tools and intermediate files from the runtime image. Q20: What are Docker layers and why are they important?

Answer (English):

Docker layers are read-only filesystem snapshots created by each instruction in a Dockerfile. Layers are cached and reused, making subsequent builds faster. Q23: What is a Docker volume and why is it used for databases?

Answer (English):

A Docker volume is persistent storage that survives container restarts and deletions. Databases use volumes to store data files, ensuring data persistence.
