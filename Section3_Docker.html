Q18: Why do we use multi-stage Docker builds?

Answer (English):

build in steps reduce final image size by excluding build tools and intermediate files from the runtime image. The build stage compiles the application, and the runtime stage only contains the compiled JAR and runtime libraries, resulting in smaller, faster, and more secure images.
