Q30: What happens when tests fail in the pipeline?

Answer (English):

When tests fail, the pipeline stops immediately, preventing deployment. The build is marked as failed, notifications are sent, and the developer must fix the tests before the pipeline can proceed. This prevents broken code from reaching production. | –ß—Ç–æ –ø—Ä–æ–∏—Å—Ö–æ–¥–∏—Ç, –∫–æ–≥–¥–∞ —Ç–µ—Å—Ç—ã –ø–∞–¥–∞—é—Ç –≤ pipeline? –¢–µ—Ä–º–∏–Ω—ã: Pipeline failure = –ø–∞–¥–µ–Ω–∏–µ pipeline (–æ—Å—Ç–∞–Ω–æ–≤–∫–∞ –≤—ã–ø–æ–ª–Ω–µ–Ω–∏—è) Fail fast = –±—ã—Å—Ç—Ä–æ–µ –ø–∞–¥–µ–Ω–∏–µ (–æ—Å—Ç–∞–Ω–æ–≤–∫–∞ –ø—Ä–∏ –ø–µ—Ä–≤–æ–π –æ—à–∏–±–∫–µ) Notification = —É–≤–µ–¥–æ–º–ª–µ–Ω–∏–µ (email, Slack) –ß—Ç–æ –ø—Ä–æ–∏—Å—Ö–æ–¥–∏—Ç: –ü–æ—Å–ª–µ–¥–æ–≤–∞—Ç–µ–ª—å–Ω–æ—Å—Ç—å:...

Q31: Why are Docker images tagged dynamically?

Answer (English):

Dynamic tagging (e.g., `${BUILD_NUMBER}`) creates unique image versions for each build, enabling version tracking, rollback capabilities, and traceability.
